Критично

1КритичноSoft-404: любой несуществующий URL отвечает HTTP 200

Запрос произвольного несуществующего адреса возвращает статус 200 OK с телом «404 / Страница не найдена» и мета-тегом noindex, nofollow. HTTP-статус 404/410 не отдаётся никогда.

GET /nonexistent-page-xyz123 → 200 OK + <meta robots="noindex, nofollow"> GET /- → 200 OK (soft-404) GET /index.php → 200 OK (soft-404) GET /wp-admin/admin-ajax.php → 200 OK (soft-404)

Последствия Google не отличает живые страницы от мусорных: тратится crawl budget, растёт число «не проиндексировано», в отчёте GSC уже 62 noindex и 50 «просканировано, но не проиндексировано», тренд растёт. noindex мешает, но не заменяет корректный статус.

Решение Для реально несуществующих адресов возвращать 410 Gone (или 404) с реальным HTTP-статусом. В Next.js — вызывать notFound() так, чтобы ответ был со статусом 404, а не 200. Убрать выдачу 200 на неизвестные маршруты.

Высокие

2ВысокийСтарые WordPress-URL не редиректятся — миграция не завершена

Сайт мигрировал с WordPress на Next.js, но старые адреса остались в индексе Google и отдаются как soft-404 (200 + noindex) вместо 301 на новые аналоги. Английские слаги заменены русскими (/vacancies → /vakansii), но редиректа нет.

/vacancies → 200 (soft-404) ожидается 301 → /vakansii /vacancies/ → 308 → /vacancies → 200 (soft-404) /feedback → 200 (soft-404) ожидается 301 → /kontakty /submit-application, /index.php, /wp-admin/admin-ajax.php → 200 (soft-404) — 410 /bank-otkazal-v-ipoteke-...-nedvizhimost (старый блог) → 200 (soft-404)

Последствия Теряется ссылочный вес со старых URL (внешние ссылки WP-эпохи ведут в никуда), краул тратится на мёртвые адреса, каждый URL считается дважды (со слэшем и без).

Решение Настроить 301 со старых слагов на новые эквиваленты; чисто технические WP-эндпоинты (/wp-admin/*, /index.php, /-) — 410. Список брать из GSC → Индексирование → noindex (62 URL, есть экспорт).

3ВысокийДва <h1> на главной

На главной два тега H1: основной заголовок и заголовок формы.

H1[0]: «Кредит под залог недвижимости в Алматы» H1[1]: «Оставить заявку» ← заголовок формы

Последствия Размывается главный тематический сигнал страницы. На подстраницах (/zalog-avto) корректно — один H1, значит проблема точечная.

Решение «Оставить заявку» перевести в <h2>/<p>, оставить один H1 на главной.

Средние

4Среднийog:image отсутствует при заявленном summary_large_image

Объявлен twitter:card = summary_large_image, но og:image нет ни на одной проверенной странице (в raw-HTML главной — 0 совпадений).

Решение Добавить og:image (1200×630) и twitter:image на все шаблоны — иначе превью в мессенджерах и соцсетях пустые.

5СреднийДублирующиеся и конфликтующие HTTP-заголовки

Заголовки задаются дважды (приложение + прокси/CDN). HSTS приходит с двумя разными значениями max-age.

Strict-Transport-Security: max-age=31536000; ... , max-age=63072000; ... Referrer-Policy: strict-origin-when-cross-origin (×2) X-Frame-Options: SAMEORIGIN (×2) X-Content-Type-Options: nosniff (×2)

Решение Оставить источник заголовков один (либо Next.js, либо прокси). Согласовать единый HSTS max-age.

6СреднийДубль JSON-LD Organization

На главной блок Organization с @id=#organization выводится дважды (типы разметки: Organization, Organization, FinancialService, BreadcrumbList, FAQPage).

Решение Удалить дублирующийся Organization — оставить один на @id.

7СреднийРазнобой trailing slash в canonical / hreflang / sitemap

Каноникализация даёт смешанные сигналы: canonical и часть hreflang — без слэша, sitemap и фактический URL — со слэшем.

canonical (главная): https://1kredit.kz (без слэша) hreflang на /kk: https://1kredit.kz (без слэша) hreflang на главной: https://1kredit.kz/ (со слэшем) sitemap.xml / факт: https://1kredit.kz/ (со слэшем)

Решение Выбрать одну форму (рекомендуется со слэшем для корня) и привести canonical, hreflang, og:url, sitemap к ней единообразно.

Низкие

8НизкийПустые/отсутствующие alt (11 из 16 изображений)

Большинство без alt — декоративные SVG-иконки (terms-icon-*.svg, about-icon-*.svg). Для них корректно alt=""; контентные картинки должны иметь описательный alt.

Решение Декоративным иконкам — пустой alt="" (или фон CSS), содержательным изображениям — осмысленный alt.

9НизкийBreadcrumbList на главной из одного элемента

Хлебные крошки на главной содержат только «Главная». Проверить полноту цепочек на подстраницах.

Решение На внутренних страницах отдавать полную цепочку (Главная → Раздел → Страница).
